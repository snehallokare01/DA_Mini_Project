## Problem Statement 
The Google Play Store dataset contains large amounts of information about mobile applications 
available on the Google Play Store, such as ratings, reviews, installs, and categories. However, it 
is difficult to directly understand patterns and trends from raw data. 
The problem is to analyze and extract meaningful insights from this dataset to understand app 
performance, user preferences, and market trends. This includes identifying popular app 
categories, studying the relationship between ratings and installs, and comparing free and paid 
applications.

available on the Google Play Store. This dataset is used to analyze app performance, 
popularity, user ratings, and trends in the mobile application market. 
• The dataset includes several attributes such as App name, Category, Rating, Reviews, 
Size, Installs, Type, Price, Content Rating, Genres, and Last Updated date. These features 
help in understanding how different factors affect the success of an application. 
• Using this dataset, data analysts can perform Exploratory Data Analysis (EDA) to

## Dataset Features: 
  App – Name of the application  
  Category – Type of app (Game, Education, Business, etc.)  
  Rating – User rating (1 to 5)  
  Reviews – Number of user reviews  
  Size – Size of the app (MB/KB)  
  Installs – Number of downloads/installs  
  Type – Free or Paid app  
  Price – Cost of the app  
  Content Rating – Suitable age group (Everyone, Teen, etc.)  
  Genres – App genre/category  
  Last Updated – Date of last update  
  Current Ver – Current version of the app  
  Android Ver – Required Android version

## Methodology 
1. Data Collection 
• The dataset is collected from online sources such as Kaggle.  
• It contains information about apps like ratings, installs, reviews, and categories.  
• The dataset helps analyze the performance and popularity of mobile applications.  
2. Data Loading 
• The dataset is loaded into Python using the Pandas library.  
• The CSV file is imported using pd.read_csv() function.  
• The data is stored in a dataframe for further analysis.  
3. Data Cleaning 
• Missing values in columns such as ratings or size are handled.  
• Duplicate records are removed from the dataset.  
• Data types are converted into the correct format for analysis.  
4. Exploratory Data Analysis (EDA) 
• Basic statistics like mean, median, and standard deviation are calculated.  
• Relationships between variables such as rating and installs are examined.  
• Patterns and trends in app categories are identified.  
5. Data Visualization 
• Graphs such as bar charts, scatter plots, and box plots are created.  
• Visualization helps understand patterns and comparisons easily.  
• Libraries like Matplotlib and Seaborn are used.
